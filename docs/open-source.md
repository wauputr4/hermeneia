# Open Source Readiness

Hermeneia is intended to be open-source friendly from the start.

## Principles

- Clear README.
- Clear license.
- No secrets in repository.
- File-based local development first.
- Modular architecture.
- Transparent AI workflows.
- Reproducible rendering inputs.

## Recommended License

Selected license:

- Apache-2.0

Apache-2.0 is selected for stronger patent protection and common open-source adoption.

## Repository Hygiene

Initial repository hygiene includes:

- `LICENSE`
- `.gitignore`
- `CONTRIBUTING.md`
- `CODE_OF_CONDUCT.md`
- `SECURITY.md`
- `.env.example`

## AI Transparency

Hermeneia should clearly document where AI is used:

- research summarization,
- brief generation,
- revision suggestions,
- image generation/editing,
- optional copywriting assistance.

Generated outputs should remain inspectable and editable.
