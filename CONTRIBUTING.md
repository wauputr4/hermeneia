# Contributing to Hermeneia

Thank you for considering a contribution.

## Principles

- Keep workflows transparent and revisable.
- Keep generated content inspectable.
- Prefer structured inputs over raw prompt-only rendering.
- Keep backend workflow logic separate from renderer-specific logic.
- Add tests for meaningful code changes.
- Update documentation and troubleshooting notes after implementation work.

## Development Direction

Hermeneia is planned as a Go backend/CLI with a SvelteKit frontend and Remotion-based video renderer.

Before contributing code, read:

- `docs/architecture.md`
- `docs/tech-stack.md`
- `docs/workflow.md`
- `docs/roadmap.md`
- `docs/troubleshooting.md`
